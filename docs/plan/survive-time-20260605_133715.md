# P2-1: SurviveTime 幽灵配置 — 实现连接年龄驱逐

**时间**: 2026-06-05 21:37 UTC
**状态**: ✅ 已修复
**文件**: pool_manager.go, config.go, model.go

## 问题描述

配置中有 `SurviveTime: 30 * time.Minute`，README 写了"连接最长存活时间"，
resource 结构体有 `createTime` 和 `updateTime` 字段。
但**整个代码库没有任何地方**读取这些字段做驱逐判断。

`shrink` 只看 channel 利用率，不看连接年龄。一条连接可以活到进程退出。

## 修复方案

在 `shrink` 方法中添加连接年龄检查，优先驱逐超龄连接：

```go
func (a *PoolManagerActor[T]) shrink(s *PoolManagerState[T]) {
    // ... 现有逻辑 ...
    
    for closedCount < shrinkSize {
        select {
        case r := <-a.sharedResources:
            // 优先驱逐超龄连接
            if s.config.SurviveTime > 0 && time.Since(r.createTime) > s.config.SurviveTime {
                a.connControl.Close(r.Conn)
                a.poolTotalSize.Add(-1)
                closedCount++
                continue
            }
            // 非超龄连接：如果还需要缩容则驱逐，否则放回
            if closedCount < shrinkSize - 过龄计数 {
                a.connControl.Close(r.Conn)  
                a.poolTotalSize.Add(-1)
                closedCount++
            } else {
                a.sharedResources <- r  // 放回
            }
        default:
            return
        }
    }
}
```

**核心思路**：
1. 逐出一批连接时，优先选超龄的（`time.Since(r.createTime) > SurviveTime`）
2. 如果超龄连接数量足够缩容目标，则仅驱逐超龄的
3. 如果超龄不够，再驱逐正常连接
4. 非目标连接放回 channel

**为什么在 Actor 中做**：shrink 已跑在 Actor 的 eventLoop 里，串行执行，无并发问题。且 `time.Now()` 调用在串行路径中安全（不涉及 workflow 的 Date.now 限制）。

## 影响范围

- pool_manager.go: shrink() 方法增强
- 配置语义从"无效果"变为"生效"

## 冲突检查

- 与现有的 channel 利用率缩容逻辑互补（先满足缩容量，优先选老的）
- 不影响扩容逻辑（checkAndAdjust 不变）
